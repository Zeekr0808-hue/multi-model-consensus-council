# 🏛️ 多模型决策委员会

**Multi-Model Consensus Council**

专为 OpenClaw 设计的"数字智库"，通过多模型独立评审与共识合成，消除单模型偏见，为复杂决策提供客观参考。

---

## ⭐ 核心特性

- 🤝 **多模型协同**：支持 3-13 个大模型同时评审
- 📊 **量化决策**：100分制评分 + 6维度加权矩阵
- 🔍 **对抗性评审**：模型间互评，快速锁定逻辑漏洞
- 🏅 **实名透明**：评委以真实模型名称显名，全程可溯源
- ⚙️ **灵活配置**：轮数、阈值、判定方式均可自定义

---

## 🚀 快速开始

### 启动方式

在 OpenClaw 中输入以下任一指令即可启动：

- "启动多模型决策委员会"
- "审议这个方案"
- "对这个方案进行决策"
- "启动决策"

### 首次使用流程

1. **自动扫描模型** → 默认选择前 3 个可用模型
2. **确认评委组合** → 可用"换模型"调整
3. **确认配置** → 用"确认配置"开始决策

### 常用指令

| 指令 | 说明 |
|:---|:---|
| 换模型 | 调整评委模型组合 |
| 改轮数 | 调整决策轮数（默认3轮） |
| 改阈值 | 调整通过阈值（默认≥90%） |
| 改判定方式 | 切换判定模式（全票/均分/多数票） |
| 增加委员 / 减少委员 | 调整评委数量 |
| 确认配置 | 确认参数并开始决策 |

## 📋 决策流程

```
第0轮：准备阶段 — 组织者汇总用户背景、需求、待审方案
    ↓
第1轮：独立评估 — 各位评委背对背独立打分（100分制）
    ↓
第2轮：收敛讨论 — 争议点二次辩论，快速收敛
    ↓
第3轮：最终辩论 — 仍未收敛的点，简化对立选项后最终表态
    ↓
生成6段式共识报告
```

---

## 📄 判定规则

| 判定方式 | 说明 |
|:---|:---|
| 全票通过（默认） | 所有评委均需≥阈值 |
| 均分通过 | 评委均分≥阈值 |
| 多数票通过 | 超过半数评委≥阈值 |

**通过阈值**：默认 ≥90%
**待决策阈值**：默认 [72%, 90%)
**否定阈值**：默认 <72%
