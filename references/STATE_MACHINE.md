# 多模型决策委员会 — 状态机规范（概念参考）

> ⚠️ **V1.6.0 说明**：本文档描述的状态机逻辑仍为概念规范，但实际执行时已**不再使用持久化状态文件**。组织者改用「轻量级状态跟踪」（会话上下文变量）来跟踪评审进度。详见 SKILL.md 「轻量级状态跟踪」章节。
> 
> 本文件保留作为状态流转的概念参考和异常处理指南。

## 状态定义（6状态）

| 状态 | 说明 | 可执行动作 |
|:---|:---|:---|
| `IDLE` | 初始状态，无进行中决策 | 接受任务 → ROUND_0 |
| `ROUND_0` | 准备阶段，组织者汇总决策申请内容 | 分发方案 → ROUND_1 |
| `ROUND_1` | 第1轮独立评审中 | 汇总报告 → ROUND_2 |
| `ROUND_2` | 第2轮收敛讨论中 | 判断收敛性 → ROUND_3 或 COMPLETE |
| `ROUND_3` | 第3轮最终辩论中（仅在ROUND_2有分歧时触发） | 汇总矩阵 → COMPLETE |
| `COMPLETE` | 决策完成，报告已生成 | 输出报告 → IDLE |

## 状态流转图

```
[IDLE] → 用户触发 → [ROUND_0]
                         ↓
                  汇总决策申请内容
                         ↓
[ROUND_1] ← 分发方案 → [ROUND_0]
     ↓
  有分歧          无分歧
     ↓              ↓
[ROUND_2]      [COMPLETE]
     ↓
  有分歧          无分歧
     ↓              ↓
[ROUND_3]      [COMPLETE]
     ↓              ↓
  汇总矩阵    输出6段式报告
     ↓              ↓
[COMPLETE] ───────→ [IDLE]
```

## 收敛判定规则

### 收敛条件

满足以下**全部条件**时，判定为收敛，跳过下一轮：

1. 所有评委的**推荐顺序完全一致**
2. 无新产生的争议点
3. 所有评委的综合评分差距在阈值范围内

### 分歧条件

满足以下**任一条件**时，判定为有分歧，触发下一轮：

1. 任意两位评委的**推荐顺序冲突**
2. 存在**未解决的反对意见**
3. 评委评分差距超过阈值范围（>20%）

---

## 异常处理

| 异常情况 | 处理方式 |
|:---|:---|
| 某评委子会话超时（3分钟） | 报告中标注"评委X超时"，不影响其他评委结果 |
| 某评委返回格式错误 | 使用前一轮结果替代，报告中注明 |
| 用户中断决策 | 保留当前状态文件，下次可恢复执行 |
